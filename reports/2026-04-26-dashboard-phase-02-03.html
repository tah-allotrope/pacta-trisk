03 / Math / Algorithm Used

Math / Algorithm Used

Interactive page logic: the PACTA page computes simple dashboard summaries from the frozen tables rather than re-running PACTA math. KPI cards use row counts, unique borrower counts, and mean match score from 02_vn_matched_prioritized.csv. Interactive charts reshape the precomputed market-share and SDA outputs into Plotly bars and trajectories filtered by sector and technology.

No new climate model math was introduced in these phases. The implementation intentionally treats dashboard/data/ as the source of truth and only adds a presentation/query layer on top.

06 / Limitations / Second-Best Alternative

Limitations / Second-Best Alternative

  • Only Phase 03 is functional today: the TRISK, Reports, and Methodology pages are deliberate stubs so the app shell exists without pretending those later phases are done.
  • Static image constraint remains: the PNG charts are upstream snapshot outputs, so the sector filter changes the interactive table and Plotly views, while the image panel only swaps between pre-rendered files.
  • Second-best alternative considered: skip Plotly entirely and ship only static PNGs plus tables. Rejected because the sector drilldown benefits materially from an interactive layer, and Plotly stayed light enough for this scope.
07 / Errors / Warnings / Flags

Errors / Warnings / Flags Encountered

  • Real issue found and fixed: AppTest initially failed because the Streamlit files imported lib as a top-level module. The fix was to add dashboard/__init__.py and switch imports to dashboard.lib.*.
  • Verification outcome: after the import-path fix, python -m pytest dashboard/tests passed cleanly.
  • Runtime flag: the short-lived Streamlit boot command timed out intentionally after confirming the app started and exposed a local URL; this was a controlled verification timeout, not an app failure.
