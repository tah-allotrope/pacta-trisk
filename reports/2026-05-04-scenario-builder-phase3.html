03 / Math / Algorithm Used

Math / Algorithm Used

Scenario ID encoding: Deterministic string key of the form s{shock_year}_d{discount_rate}_rf{risk_free_rate}_mp{market_passthrough}_c{carbon_price_family} enabling O(1) lookup into the precomputed grid.

Rank delta: Baseline rank minus scenario rank. Positive = borrower moved up in priority (worsened relative to peers). Computed via stress_priority_score.rank(ascending=False) independently for baseline and scenario, then diffed by company_id.

Top movers: Sort by absolute delta_npv_change_vs_base and delta_pd_change_vs_base, take top 5. Shows which borrowers are most sensitive to the specific lever combination.

Grid lookup: Precomputed grid contains all 3³ x 3 = 243 combinations (3 values each for shock_year, discount_rate, risk_free_rate, market_passthrough, carbon_price_family). No interpolation between grid points — snap to nearest grid cell.

06 / Limitations / Second-Best Alternative

Limitations / Second-Best Alternative

  • Discrete grid only: Lever values snap to the precomputed 3-value grid. Continuous sliders are not available unless the operator live-rerun adapter (PHASE-04) is enabled.
  • Single-session save: Saved scenarios exist only in st.session_state and are lost on page refresh (different from deep-link restore). Cross-session persistence requires auth and storage.
  • No live R execution: The public deployment path uses the precomputed grid only. Custom parameter combinations outside the 243 predefined cells are not available on pactavn.streamlit.app.
  • Baseline from sensitivity: The baseline view uses the base rows from sensitivity_results.csv, which may not perfectly align with the grid's own implicit base used for delta_*_vs_base columns.
  • DRY tradeoff: The page duplicates some logic (scenario_id parse/build, grid label formatting) that could be shared with a future common module. Kept inline per plan's simplicity-first constraint.

Second-best alternative: Build the page against a 2-value grid (32 cells instead of 243). Rejected because 3-value grids are still within runtime budget and provide visibly meaningful "no change" vs "material change" comparisons during demos.

07 / Errors / Warnings / Flags

Errors / Warnings / Flags Encountered

  • Scenario ID parse edge case: Initial split("_") approach failed for carbon_price_family values containing underscores (e.g. NGFS_NetZero2050). Fixed by using split("_c", 1) to isolate the carbon family suffix.
  • TRISK page smoke test timeout: test_trisk_page_renders and test_trisk_page_allows_sector_switch in test_smoke.py time out at 3s default. This is pre-existing (not caused by this phase). Mitigated by adding explicit timeout=15 to at.run() calls.
  • pytest-timeout not installed: @pytest.mark.timeout markers produced unknown-mark warnings. Removed markers since the inline Streamlit timeout parameter handles it.
  • Chart deprecation warning: Plotly use_container_width will be removed after 2025-12-31. Replace with width='stretch' at that point.
