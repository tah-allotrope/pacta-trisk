4. Math / Algorithm Used

  • Power remains on the existing market-share pathway logic: baseline production is derived from capacity and capacity factor, then scaled by `tmsr` / `smsp` from `data/vietnam_scenario_ms.csv`.
  • Cement and steel use SDA emission-intensity pathways from `data/vietnam_scenario_co2.csv`.
  • For cement and steel, `scenario_price` is a synthetic unit-price curve increased by the intensity reduction required versus the sector baseline.
  • `scenario_pathway` and `scenario_capacity_factor` are derived from the ratio between target intensity and the sector baseline intensity, with different multipliers for `carbontech` and `greentech` technologies.

7. Limitations / Second-Best Alternative

  • Phases 1 and 2 stop at input generation; no sector TRISK runs or dashboard changes landed yet.
  • Cement and steel still rely on synthetic scenario translation and will need clear labeling in later phases.
  • Second-best alternative not chosen: separate `trisk_cement_demo.R` and `trisk_steel_demo.R` clones. That would be faster short term but would duplicate the power-only assumptions instead of centralizing them behind one sector contract.

8. Errors / Warnings / Flags Encountered

  • Environment warning: `Rscript` is not on PATH, so verification had to use the full executable path.
  • Resolved bug 1: the first refactor wired `purrr::imap()` argument order incorrectly, which produced an invalid output path.
  • Resolved bug 2: the first cement/steel aggregation attempted to reuse a just-summarized column as its own weight vector, which broke weighted means for sectors with multiple borrowers.
