03 / Math / Algorithm Used

Math / Algorithm Used

Subprocess contract: The R script prints exactly one line to stdout: the temp CSV path. All diagnostics (progress, timing) go to stderr. The Python adapter reads the last stdout line as the path, loads the CSV, and deletes it.

Carbon price family mapping: NGFS_NetZero2050, NGFS_Below2C, and NGFS_Delayed are each mapped to the sector’s existing carbon_price_model string (e.g. increasing_carbon_tax_50 for power, cement_intensity_transition for cement). All three families use the same underlying curve for v1; the distinction is narrative only.

Time budget: The R subprocess has a 30-second wall-clock timeout. A single TRISK run typically completes in 2-5 seconds on a developer laptop, leaving headroom for load and garbage collection.

06 / Limitations / Second-Best Alternative

Limitations / Second-Best Alternative

  • R dependency: The live-rerun path requires R and trisk.model installed locally. This is by design — the precomputed grid path (PHASE-03) serves Streamlit Cloud which has no R runtime.
  • No delta normalization: The live rerun output uses stress_priority_score directly from summarize_trisk_run(), which may differ in scale from the grid’s normalized scores. The side-by-side comparison shows absolute values, not deltas.
  • Single concurrency: Only one live rerun per session at a time. This is intentional to avoid trashing the local R process.
  • No carbon price curve toggle: All three carbon_price_family values map to the same underlying model string for each sector. True NGFS curve differentiation is a follow-up.
  • No caching across sessions: Live rerun results are ephemeral — they exist only in that session’s state until page refresh.

Second-best alternative: Skip the R subprocess entirely and interpolate between grid points in Python. Rejected because interpolation of TRISK outputs (NPV/PD) is non-linear and misleading; only the R model produces valid results for arbitrary parameter combinations.

07 / Errors / Warnings / Flags

Errors / Warnings / Flags Encountered

  • Pre-existing smoke test timeout: test_trisk_page_renders continues to time out due to data loading latency in the Streamlit AppTest harness. Not caused by this phase.
  • Windows CRLF warnings: Git warns about LF/CRLF conversion on commit. Normal for Windows development.
  • R subprocess portability: The R script uses tempfile() and Sys.getenv(), both of which are cross-platform. Tested on Windows; should work identically on Linux/macOS.
