05 / Results & Visualizations

Results & Visualizations

Verified Results

  • All three sectors now publish `scenarios.csv`, `borrower_results.parquet`, and `grid_meta.json` under both synthesis output and dashboard snapshot paths.
  • `dashboard/data/trisk/manifest.csv` now reports `grid_available=TRUE` for `power`, `cement`, and `steel`.
  • The generator can rerun incrementally because cached run folders are reused instead of recomputed.
  • The public snapshot path remains R-free even though the grid build itself still happens in R offline.
06 / Limitations / Second-Best Alternative

Limitations / Second-Best Alternative

  • The generator currently publishes the full `runs/` and `input/` subdirectories into the dashboard snapshot grid path. That is acceptable for now but can be trimmed later if snapshot size matters.
  • The three carbon-price families still map onto existing curves, so scenario-family differentiation is naming-level rather than curve-level in v1.
  • The old topic worktree could not be fully deleted automatically because of a Windows file lock; the merged implementation itself is unaffected.
  • Second-best alternative would have been to reduce the grid or skip `2030`, but extending the grid-only inputs was the cleaner fix because it preserved the approved contract.
07 / Errors / Warnings / Flags

Errors / Warnings / Flags Encountered

  • Missing R package: `arrow` was not installed initially; installed it into the user library to support parquet output.
  • Implementation bug: the first grid build failed because `sector` was shadowed inside `mutate()`, breaking carbon-price model mapping.
  • Model/runtime bug: `shock_year = 2030` failed inside `trisk.model` because the synthetic inputs did not extend far enough for interpolation.
  • Windows file-lock issue: cached parquet rewrites failed until the generator was changed to skip complete sectors and use a temp-file swap.
  • Non-blocking warnings remained from package build versions and TRISK's Merton-model row filtering.
